## Design Context

### Users

Tournament arbiters and club directors running chess events — from casual weekly
club nights to FIDE-rated tournaments. They range from volunteers with basic
computer skills to experienced arbiters who've used dedicated tournament
software for years. They use the app on a laptop at the tournament venue, often
under time pressure between rounds. Reliability and speed matter more than
visual flair.

### Brand Personality

**Clean, efficient, dependable.** Like a professional tool — no-nonsense, gets
the job done. Should evoke professional confidence while remaining approachable
enough that a first-time club director can pick it up without training.

### Aesthetic Direction

- **Visual tone**: Professional sports scoring — think broadcast scoreboards,
  FIFA match centers. Authoritative but readable. Data-dense when needed, clean
  when not.
- **Theme**: System-aware (light and dark), but currently dark-only.
- **Anti-references**: Should NOT look like a chess.com clone (current palette
  is literally labeled "Chess.com Dark Palette"). Should NOT look like a generic
  shadcn starter template.
- **References**: Professional scoring systems, ESPN match centers, native macOS
  productivity apps.

### Design Principles

1. **Clarity over decoration** — Every visual element must communicate, not
   decorate. An arbiter glancing at the screen between rounds needs instant
   comprehension.
2. **Confidence through restraint** — Professional tools earn trust by being
   predictable and quiet, not flashy. The interface should feel like it knows
   what it's doing.
3. **Data is the hero** — Player names, scores, pairings, and standings are the
   content. The UI should frame and organize this data, never compete with it.
4. **Progressive complexity** — Simple tasks (start tournament, enter result)
   should be obvious. Advanced features reveal themselves through use.
5. **Own identity** — Kx8ble is not chess.com, not Lichess, not a generic
   template. It should have a distinctive, recognizable identity appropriate for
   tournament management software.
